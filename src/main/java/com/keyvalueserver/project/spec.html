請依下面需求設計 API端點和操作 1. 獲取鍵值（Get key） Endpoint: GET /keys/{key} 功能: 返回與指定鍵相關的值。 參數: key (在URL中): 要獲取的鍵。 2. 設置鍵值（Set key Value） Endpoint: POST /keys 功能: 為指定的鍵設置值。 請求體: key: 要設置的鍵。 value: 與鍵相關聯的值。 3. 刪除鍵（Delete key） Endpoint: DELETE /keys/{key} 功能: 刪除指定的鍵和其值。 參數: key (在URL中): 要刪除的鍵。 4. 將所有鍵值對下載為CSV檔案 Endpoint: GET /keys/download 功能: 下載包含所有鍵值對的CSV檔案。 返回: CSV檔案。 5. 身份驗證（使用JWT Token） 實現方式: 對所有API請求實施JWT身份驗證。 在請求頭中添加一個 Authorization 欄位，其值為 Bearer ，其中 是JWT。 JWT Token驗證流程 用戶首先通過一個身份驗證端點（例如 POST /auth/login）使用其憑證登錄。 伺服器驗證憑證，並發放一個JWT。 用戶在後續請求的 Authorization 頭部中包含此JWT。 伺服器在每次請求中驗證JWT的有效性。 注意事項 所有的數據交換格式建議使用JSON。 確保API的安全性，尤其是在處理JWT和用戶數據時。 錯誤處理應詳細，並提供清晰的錯誤信息。 實作注意 1. 使用spring boot框架來做http 2, 使用Map來儲存key和value 3. 會有一個Backgrond thread來把異動寫入到外部MySQL(備份整個map) 4. 部署到cloud run 序列圖 1.Get Key
